This is the living map of the platform: where everything is, what rules apply.
**Every worker (main session or background agent) reads this before touching the
repo and updates the sections it changed after shipping.** Keep it a MAP, not
documentation — if a section grows past a screen, cut it. History and lessons

_Last full verification: 2026-07-14_

## The goal
Clone a sales/CS rep from their note-taker calls — voice, speech style, screen
behavior, call flow — into an agent that runs real Zoom demos of GoPerfect.
Convergence should cost machine-iterations, not founder-hours: links in →
observed blueprint → overnight fidelity runs → ~1h of human coaching → pinned
golden → certified → real calls that feed the loop. Target: new rep cloned in
<1 day elapsed, <1h operator attention.

### Data flows (the five that matter)
1. **Clone lifecycle**: share links → transcripts+urls → persona extraction (ground truth only, kind≠live_call) + auto-storyboard → calibrate (every edit = new version; hot-reloads a RUNNING call via pushPersonaReload) → pin golden (only golden drives real calls; playbook changes recompile golden automatically) → certification → workspace.
2. **Call pipeline**: join → E2B sandbox → chrome kiosk → gp_login → bridge (AH_AGENT_ID scopes golden; curtain until first nav) → feed/nudges/coverage → end → transcript saved as live_call source + auto-debrief.
3. **Voice**: agents.voice_id is authoritative everywhere → room Talk (shared ConvAI agent, per-session overrides, WebRTC for echo cancellation) → live calls (hybrid: realtime text → EL flash stream → pacat; kill-switch settings call_voice_mode='openai'; falls back to marin, never mute) → "— real voice" cloned from Fathom HLS media.
4. **Site map**: cartographer crawl (own sandbox; catch-all fingerprint probe; structure signatures; dynamic detection) → settings site_map → bridge goto (structure-first verify for dynamic pages, requires→satisfyWith prerequisites) + Storyboard card.
5. **Fidelity**: observe-screens (HLS keyframes ~13s → vision labels vs site-map vocab → observedTimeline+timed turns on clone_sources.observed; creates the source if the share is new) → enrich-playbook (re-timed/corrected DRAFT, save stays human via playbook PUT) → fidelity/run (replay real customer as guest nudges on a rehearsal → per-moment LLM diff vs human reply+observed screen → auto-fix ≤5 via versions/playbook endpoints → settings fidelity_report:<id>; forces call_voice_mode=openai, restores after; GET /api/fidelity/latest).
